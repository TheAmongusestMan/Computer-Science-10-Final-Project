Tips


A few tips I can provide for the unity assets bar is to always create an empty gameboject before incorperating all the assets
going into the gameobject as it serves as a pseudo-folder and makes it way easier for organisation and debugging.
You should also import sprites from outside unity when possible as creating in-house sprites not only hog the storage
and operation space, but also are a PAIN to edit and layer one by one.
You should also order the assets bar when nessicary, grouping enviroment asets under one empty gameobject
and player/enemy characters/gameobjects under a likewise gameobject.

Animations should always connect to an idle or defalut animation and the initial "parameter" connection
path should not have an exit time (but the exit/back to defalut path should, and it should be 1), the transition
time should be 0 for instantanious animation. Animations will not change the hitbox of your charachter
so don't make the comat system rely on animations to work. Instead you should just create ultiple points around the player
rig for the combat to take place or interact with the player. (combat points, points where player can damage others)
